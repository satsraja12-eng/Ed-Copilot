Layer 2 — LangGraph Orchestrator
🔀 StateGraph — EdCopilotState
Stateful LangGraph graph; the state object carries all data across nodes. Compiled once with @st.cache_resource.
Node — classify_intent
Lightweight LLM call — reads last user message, returns one of four intent labels. Writes intent + intent_badge to state.
State Fields — EdCopilotState
messages[] · persona · district
intent · context_docs[]
response · intent_badge
Conditional Edges
Routes on state["intent"] to one of four specialist branches below. All branches merge back to state["response"].
Layer 3 — Specialist Nodes
📐 math_specialist
Handles intent = math_curriculum. Calls the hybrid retriever on chroma_db/. Persona-aware prompt shapes answer style.
Persona prompts:
Student — simple + encouraging · Parent — practical context · Teacher — precise, full standard ID
🏫 admin_specialist
Handles intent = admin_policy. Retrieves from chroma_db_admin/ with a district metadata filter so districts never cross-contaminate.
Sources expander: shows source_url + label per retrieved chunk · District disclaimer appended to every response
🎓 guidance_stubPhase 2
Handles intent = college_guidance. Currently returns a planned-phase stub message. Full implementation in Phase 2.
Phase 2 plan: Student Profile → Articulation Engine → Prerequisite Graph → chroma_db_guidance/ → LLM
🚫 out_of_scope_handler
Handles intent = out_of_scope. No LLM call. Returns a hardcoded redirect message immediately.
Layer 4 — Retrievers & Knowledge Bases
🔍 Hybrid Retriever
ChromaDB dense BM25 sparse Cross-Encoder reranker
Collection: chroma_db/ — 1,158 chunks from NC Math 1, 2 & 3 PDFs, chunked by NC Standard ID (regex split on NC.M<n>.xxx markers). Metadata: standard_id, course_id.
🗂️ Admin Retriever
ChromaDB similarity_search district metadata filter
Collection: chroma_db_admin/ — web pages + policy PDFs from all three districts. Metadata filter {"district": district} scopes every query to the selected district. Top-5 chunks returned.
🗃️ Guidance RetrieverPhase 2
chroma_db_guidance/ domain=guidance
Phase 2 sources: WCPSS / Frisco / Plano program planning guides, university admissions pages (UNC, NC State, UC Berkeley, UC San Diego), Wake Tech CCP catalog.
⛔ No Retrieval
Out-of-scope path skips the retriever entirely. Hardcoded response only.
Layer 5 — Shared LLM Infrastructure
🤖 Nebius / Llama-3.3-70B-Instruct
Single shared LLM instance used by classify_intent, math_specialist, and admin_specialist. Accessed via the OpenAI-compatible Nebius API endpoint.
🎭 Persona-Aware Prompt Modifier
Injects a persona instruction into every specialist system prompt before the LLM call.
Parent — practical context
Student — simple + encouraging
Teacher — precise + full detail
Layer 6 — Ingestion Pipelines
⚙️ Math Ingestion (src/ingestion.py)
PyMuPDFLoader → regex split on NC Standard IDs → per-standard chunks → ChromaDB
Chunking: Regex pattern NC.M<n>.xxx — one chunk per standard. 1,158 chunks in chroma_db/. Unchanged from current implementation.
⚙️ Admin Ingestion (src/admin_ingestion.py)
WebBaseLoader (web pages) + PyMuPDFLoader (policy PDFs) → RecursiveCharacterTextSplitter (800 tokens, 100 overlap) → enrich metadata → ChromaDB
Dedup: Skips URLs already indexed today. Config: config/admin_sources.yaml (Option 3 format — future-ready for hybrid discovery).
⚙️ Guidance IngestionPhase 2
Program planning guides (PDF) + university admissions pages (web) + Wake Tech CCP catalog → chroma_db_guidance/
Metadata: domain=guidance, district, source_type (local_curriculum / university_admissions / dual_enrollment)
—
No ingestion pipeline for out-of-scope path.
